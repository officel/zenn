# tl;dr

- 2026年3月16日（月）15時ごろから自宅のインターネット回線が不通になり業務に支障がでた
- 2026年3月17日（火）18時になっても解消しなかったため丸一日業務ができなかった
- 2026年3月18日（水）16時ごろNTTから解消した旨の連絡があった
- 18日から19日まで家族旅行を予定していて、図らずも大きな春休みになってしまった
- ポストモーテムとして記録し、今後の対応について整理した

# 時系列

## 2026年3月16日（月）

- 9時から15時までは繋がっていた
- 家のネットワークが遅いのはいつものことなので多少の遅延は気にしていなかった
- 15時過ぎごろ（正確な時間はわからない）から断続的にウェブサイトに繋がらない状態になった
- 17時には完全にネットワークが切れてしまい、slack と teams から追い出された
- 後述する動作確認等を行ったが解消せず、一時的なものなら翌日には解消するかも、と諦めた

## 2026年3月17日（火）

- 6時に起床し回線を確認、改善されておらずネットワーク不通
- インターネットプロバイダーの問題ではなく、ルーター以降（外）の問題と判断、NTT障害情報等をスマホで確認
- 問い合わせ成功後、メールによる障害情報確定
- 結局、終日解消しなかった

## 2026年3月18日（水）

- 以前から家族旅行を計画していて仕事は休みにしていた。予定どおり出発
- 16時ごろNTTから携帯に着信し、修理完了の連絡をいただいた
- 手短に聞いた話によると自宅（集合住宅）の構内配線部分に故障があり実は全戸障害だったらしい
- なにかしらの補填があるわけでもなくすいませんで終わったので、やはり予備回線の準備をしようと決意

## 2026年3月19日（木）

- 旅行から帰宅後、PCの電源を入れたところ無事ネットワークが復旧しているのを確認した

# 対応その他

## 家庭内での確認

- PC の停止・起動（Windowsあるある）
- 電話機・ルーター（VH-100E）・Wi-Fiルーター（Buffalo Air Station）の電源オフ（コンセント引っこ抜きｗ）
- Air Station の設定とログ確認
- 結果、ルーターに届いているのは間違いないが、その先からの応答があったりなかったりした
- インターネットプロバイダーの障害情報をスマホで確認したが（いつものことだけど）パブリックには発信されていなかった

## NTT東日本

- スマホから検索して問い合わせ窓口をあれこれさまよった
- 問い合わせの送信に10回近くかかった（問い合わせが成功するまで何度も情報入力をさせられた。おかげで自分の顧客番号を覚えてしまったｗ）
- 問い合わせの送信成功後、チャットでの連絡を選択したが10分以上待たされて結局繋がらなかった
- 問い合わせの連絡待ちを選択したところ、通信状況の確認という機能に繋がった
- 電話番号を入力して待たされること10分、NTT側から我が家のルーターに接続できない旨の表示がでた
- メールによる問い合わせの自動応答により、NTT側の障害であると確定した
- 解消後に担当者から連絡するという内容だった
- 前述のとおり18日16時ごろに解消した旨連絡があった

# 振り返り

## よかったこと

- 本当の原因がNTT側にあることを確認できたので、あわててルーター類を買い替えなくて済んだ
- 18,19日と家族旅行を予定していたが、事前準備は完全に済ませてあったので困ることがなかった

  - 旅行の日程調整や情報整理は GitHub でやってる。これは別途記事にしたいところ
  - JR（新幹線）の切符はスマートEXを使っていて、交通系ICカードの登録は14日までに済ませていた。これは本当に助かった。事前準備大事

- リモートワーク中心になって長いが、予備回線の重要さがわかった
- ノートPC（と移動通信のための何か）があれば回避できたはずなので、購入のための家庭内稟議が通る（通すｗ）
- ブログのネタになったｗ
- なにより（スマホから） slack によってチームへの連絡は支障なくできたのはよかった（大事）

## よくなかったこと

- ほぼ2日間、仕事にならなかった（チームのみんなには迷惑をかけてしまい申し訳ない。。。）
- 個人用のスマホは現職のツール類について slack のみ登録していて、他のツールが一切使えなかった
- 前述のとおり予備回線の準備がなく、インターネット接続がなくなると何もできない
- ノートPCや移動通信のための何かがあれば、仕事に穴をあけることはなかったはずだ

# できなかったこと

ポストモーテムとは違うが、できなかったことのリストを作ることで、今後の改善等を検討する元ネタにする。
スマホで対応できた分（slack）については置いておく。業務だけじゃなくて個人的な作業も含む。

## 会議等

- slack はかろうじてスマホでアクセスできるようにしていたので連絡はできた
- MS Teams は現職でしか使用しておらず、スマホにはインストールもしていなかったので、会議に一切参加できなかった（本当にすまんかった）
- いったんスマホに Teams アプリを入れて開けるようにはした

## terraform

- そもそも terraform コマンドのほとんどが死ぬ
- 現職は HCP Terraform を使用しているが、インターネット通信がないと本当になにもできない
- 普段の業務のほとんどが IaC なのでネットワーク障害は致命的
- それでもノートPCがあればコワーキングスペース等で作業はできたはずだ

## git, pre-commit

- `git fetch` ができないので最新化できない
- もちろん `git push` もできない
- pre-commit を設定しているリポジトリではアクションのバージョンアップができなくて困った（タイミング！）
- とりあえずコミットは `-n` で pre-commit を避けて対応した（後で処理しなおす）
- コードを書くこと自体はできるが、`fetch` も `push` もできないのは仕事にならない
- この記事の下書きや個人作業は置いておけばいいが、仕事にはならない

## GitHub, Azure DevOps

- GitHub のスマホアプリは入れていて普段から利用しているので、個人的な使用自体は問題なかった
- Dependabot が流してくるPRのチェックが若干つらかった
- 現職は GitHub EMU を使用しているが、スマホアプリで使えるようにしていなかったために何もできなかった
- 同じように Azure DevOps も対応する準備をしていなかったので何もできなかった
- 業務のPRがチェックできないのはつらい

## SSO（というか chrome profile と仕事用のアカウント）

- 業務委託として社外の人間として、個人用の設定と仕事用の設定はできるだけわけるようにしている
- 現職はSSOで大部分のSaaSにアクセスできるようにしてあり、デスクトップPCで設定済の Chrome ではプロファイルをわけているおかげで混ざることなく使い分けができている
- スマホには仕事用のプロファイルを設定していなかったため、業務用SaaSをまったく利用できなかった
- 業務用の作業環境も予備を用意しておく必要性を強く感じた

# やること（できるようにしておくこと）

突然家庭用ネットワーク回線が使えなくなった場合でも、準備しておくことで対応可能になる・できることがあることがわかった。今後のことも鑑みてできるようにしておくことを挙げる。

## 予備回線

- もしかしたら普段からノートPCを持ち歩いている人には普通なのかもしれないが、予備の回線として移動無線の準備が必要
- モバイルルーター等について今まで必要としていなかったため情報収集が必要
- 無線のあるコワーキングスペース等にあたりをつけるだけでもいいかもしれない
- でも俺、普段からほんと家から出ないんだよね。。。

## 予備ノートPC

- 基本的に家で仕事をしているため、ノートPCの導入を見送ってきた
- 前述のとおり、回線は予備を検討しているが、デスクトップPCもいつ故障するかわからない
- お安くないので後回しにしていたが、常にどこでも仕事ができるようにノートPCを用意しようと思う
- 家からは出ないけど、ノートPCがあればこたつのある部屋に移動して作業できるね！ｗ

## スマホ対応

- 今まで一度も必要になったことがないので軽く見ていたが、業務で使用するツールに対して可能な限り対応準備をしておくのは大事
- とくに今回は MS Teams に対応できていなかったことで、いくつも会議をすっぽかすことになってしまったのはいただけない
- 予備ノートPCによって軽減されると思うが、業務に必要なツール類はなるべく代替できるように準備をおこなっておく必要がある
- 良し悪しや意見の割れるところではあるかもしれないが、円滑に業務を遂行するには代替機器とその設定は重要だと思った

## 障害対応訓練

- 普段当たり前にできていることも、突然できなくなることがあることが身に沁みた
- 今後は定期的に振り返りとチェックリストの更新を行い、障害対応訓練をしていこうと思った
- 家庭内カオスモンキーかな

# 最後に

- SRE として当然のことだと思っていたが、いざ自分に降り掛かってみないとわからないというか、対応できていないことによる問題が自分ごとにできていなかった（そのために仕事に穴を開けてしまった）
- 自宅作業でリモートワーク、デスクトップPCのみでノートPCを持っておらず、移動通信機器も所持していない、というのもかなり限定された身分なのかもしれないけど、万が一に備えて準備していきたい
