# やはり俺の VS Code 設定は間違っている

- VS Code が重いのは節操なしに機能拡張を突っ込んだり、ホストに見合わない性能を求めるせいじゃないの？
- settings.json をクリーンにしておくことで余計な負荷を下げられる？
- なくても困らない機能拡張はアンインストールする
- 最終更新日が古いものにも注意する
- 使用不可になった設定は削除する
- バージョンアップで本体の機能面でも変わっていくので対応していく
- 俺は本物がほしいｗ

# 信じていたバックアップ設定にダンジョン奥地で殺されかけたがgit『checkout』でレベル9999の仲間達を手に入れて元設定と失敗設定に復元&『ざまぁ！』します

- やり直せばどうとでもなるだろって気軽に機能拡張をアンインストールしてたらまだ必要な設定が消えた
- VS Code のバックアップ設定ができていなくて戻せない
- こんなこともあろうかとシンボリックリンクを貼って別のリポジトリに保存してあった
- git はレベル9999のエリー様の魔法くらい役に立つｗ
- わたくしは今ここに「設定複数保存主義」を掲げさせていただきますのｗ
- → 最終的に復元した設定も含めて、実際そんなに要るっけ？ってなったので結局全部外してしまった

# 押さないキーボードショートカットとはラブコメにならない

- あとからあとから~~幼馴染~~機能拡張を追加して、一度も押したことのないショートカットが増えていく
- どれか~~幼馴染~~機能拡張抜いたら急に使えなくなるショートカット（主にキーバインド系はあとから抜くとツラい）
- やりたいことを整理しつつキーボードショートカットを整理する
- どれがいいのか決められないッ
- 機能拡張もハーレムがイイデスｗ
- とかネタで書いてたらいつの間にかオンリーワンになりたがっている、だと。。。
- ところでなんで keybingings.json のスキーマって name とか description とかないんだろうね？

# Re:ゼロから始める VS Code 設定

- 機能拡張や設定の入れ替えをリセットしたり試したりするのは profile が一番楽（たぶん）
- 機能拡張はすべてのプロファイルに機能拡張を適用するにチェックが入っているとついてきちゃう（記憶があるッ）
- 壊れたらやり直せばいいし、より新しくていい方向にできる可能性もあるのでがんばって試す
- ここ（profile）から始めましょう、イチからいいえ、ゼロから！
