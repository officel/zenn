# 前提と方針

- Windows や Mac など、チームメンバーに環境差がある
- チームメンバーの技術や知識にレベル差がある（守備範囲が違うというだけ）
- ツールの選定はたいていの場合レゴブロックのようなものです
- できるだけムダを省いて同じような環境を構築できるようにしつつ、自己責任で入れ替え可能にする（交換容易性を確保する）

## 対象

- コードの品質に自信があるとは言えない
- 品質ゲートが欲しい
- 技術選定に迷っている
- そんな方々向け
- あと現職の同僚氏たちに捧げますｗ
- 基本的に各自のローカルから処理可能にするためのチーム向け設計がされています

## 対象外

- 完全初心者や非エンジニアは対象外です。ゼロから説明はしていません。細かいことはまわりの人に聞いてください
- 上級者も対象外です。前述のとおり交換容易性を重要視しているので気に入らなければ好きに交換してください
- DevContainer や GitHub Codespaces は今回は対象外です

# 完成品の説明

- git リポジトリを GitHub でホスティングして、ワークフローを任せます。git がすべての中心です
- git の pre-commit hook を、pre-commit.com 互換の prek に任せます
- ツールの管理に aqua を使用します
- git commit 時にチェックを走らせます（`git commit -n` で避けることもできます。するな）
- GitHub で PR を作成したら CI として同じチェックを走らせます（ワークフローで自動化）
- CI として pre-commit が通っていれば品質ゲートとして機能するので PR 時のレビューが楽になります（なるといいな）

# git と GitHub

- git も別に再度記事を書く予定。とりあえず `git branch -d main` していこｗ
- git は最新じゃなくても問題ないと思いますが定期的に更新する癖は作りましょう
- GitHub をもっと活用しましょう
- GitHub Actions に任せていいところはできるだけ任せましょう
- Dependabot に任せていいところはできるだけ任せましょう、どうしてもって理由がない限り Renovate は要りません
- GitHub の設定でガードレールをしいて、できること（やっていいこと）とできないこと（やってはいけないこと）を整理しましょう

# aqua（ツールの管理）

- ツールの管理には [aqua](https://aquaproj.github.io/) を使用します
- 各自好きな方法でインストールしてください
- 一般的なユーザーはバージョンマネージャーを最新以外で使う必要がないので、Mac と WSL なら brew が楽です
- リポジトリに配置する `aqua.yaml` に書かれたツールが用意できるなら他のツールを使っても問題ありません
- `aqua-cheksum.yaml` を配置している場合は、それに従えるツールを使うのが楽です
- mise は aqua で管理している設定を内包できるので、mise がよければそれでも構いません

:::message
後述する prek が使用するツールを aqua で管理するので、aqua がインストールされていれば後は自動化できます
:::

## リポジトリに置く aqua.yaml

- `aqua init` すれば aqua.yaml が置かれる
- 下記のように `checksum` を有効にして運用するのがセキュリティ的に望ましい（てゆうかしとけ）

```shell
$ cat aqua.yaml
---
# yaml-language-server: $schema=https://raw.githubusercontent.com/aquaproj/aqua/main/json-schema/aqua-yaml.json
# aqua - Declarative CLI Version Manager
# https://aquaproj.github.io/
checksum:
  enabled: true
  require_checksum: true
  supported_envs:
  - all
registries:
- type: standard
  ref: v4.515.0  # renovate: depName=aquaproj/aqua-registry
packages:
```

- 使用するツール類で必要なものがあれば `aqua generate -i` でセットしておけば、aqua が自動でバージョンを切り替えてくれる

# prek（git hook manager）

- git の pre-commit hook を手で管理するのはやめましょう（.git/hooks/ に手でコードを書くなということ）
- git hook を管理するツールを git hook manager などと言い、多くのツールがあります
- [prek](https://prek.j178.dev/) を使用します
- pre-commit.com が主要ツールとして Dependabot のエコシステムに入りました。prek は同じ設定ファイルをそのまま使用できて、より高速かつオープンなので今は prek を使用するのがいいという判断です
- husky 等同様のツールは他にも多数ありますが、ほとんどが言語に依存しているので対象外です（JS/TSの開発をしていないのに husky は必要ない）
- prek 自体のインストールは前述の aqua で管理できますが、各自がシステムワイドにインストールするようにして、個別に管理しないほうが、バージョンアップ等の手間がかからずに便利かもしれません（現職では各自インストール）
- 設定ファイルとして `prek.toml` が使えますが、前述のように dependabot は pre-commit の設定ファイル `.pre-commit-config.yaml` を自動で処理するので、強いポジションがなければ `.pre-commit-config.yaml` を使用しましょう。そのままで prek でも pre-commit でも好きな方が使えます

## リポジトリに置く .pre-commit-config.yaml

- `prek sample-config > .pre-commit-config.yaml` するとサンプルファイルが作れる

```shell
# See https://pre-commit.com for more information
# See https://pre-commit.com/hooks.html for more hooks
repos:
  - repo: 'https://github.com/pre-commit/pre-commit-hooks'
    rev: v6.0.0
    hooks:
      - id: trailing-whitespace
      - id: end-of-file-fixer
      - id: check-yaml
      - id: check-added-large-files
```

- 最低限こんな感じにする

```shell
default_install_hook_types:
  - pre-commit
  - commit-msg

repos:
  - repo: https://github.com/pre-commit/pre-commit-hooks.git
    rev: 3e8a8703264a2f4a69428a0aa4dcb512790b2c8c  # frozen: v6.0.0
    hooks:
      - id: detect-private-key
      - id: end-of-file-fixer
      - id: trailing-whitespace

  - repo: https://github.com/gitleaks/gitleaks
    rev: 83d9cd684c87d95d656c1458ef04895a7f1cbd8e  # frozen: v8.30.1
    hooks:
      - id: gitleaks

  - repo: https://github.com/rhysd/actionlint
    rev: 914e7df21a07ef503a81201c76d2b11c789d3fca  # frozen: v1.7.12
    hooks:
      - id: actionlint

  - repo: https://github.com/zizmorcore/zizmor-pre-commit
    rev: a4727cbbcd26d7098e96b9cb738169b59711ae51  # frozen: v1.24.1
    hooks:
      - id: zizmor

  - repo: https://github.com/compilerla/conventional-pre-commit
    rev: 91ab4bf57e58b32adf1a122681f6ebe164d081c8  # frozen: v4.4.0
    hooks:
      - id: conventional-pre-commit
```

- conventional-commit を commit-msg hook で有効にする
- pre-commit 以外のフックはインストール時にオプションを付けないと入らないので `default_install_hook_types` を有効にしてある
- detect-private-key で秘密鍵などをコミットしてしまうのを防止する
- end-of-file-fixer でファイル末尾の改行を強制する。PR で ⛔ が差分にでなくなる
- trailing-whitespace で行末スペースの除去を強制する。余分なスペースやマークダウンの2スペース改行を禁止する
- gitleaks で機密情報の漏洩を防止する
- actionlint と zizmor で GitHub Actions のワークフローファイルを静的解析する

# 実際にやるべきことをコマンドで書くと

- 使用するPCで1回だけ

```shell
# aqua
$ brew install aqua
## aqua のグローバル設定をしておく
export AQUA_GLOBAL_CONFIG="お好きなパス"
## 一応ドキュメントの例
export AQUA_GLOBAL_CONFIG=${AQUA_GLOBAL_CONFIG:-}:${XDG_CONFIG_HOME:-$HOME/.config}/aquaproj-aqua/aqua.yaml
## 個人的にはこう
export AQUA_GLOBAL_CONFIG=${XDG_CONFIG_HOME:-$HOME/.config}/aqua/aqua.yaml

# prek
## aqua で（リポジトリローカルに）インストール（j178/prek を探して選択）
$ aqua generate -i
## aqua でグローバルにインストール（j178/prek を探して選択）
$ aqua generate -i -o $AQUA_GLOBAL_CONFIG
```

- 使用するリポジトリ毎に1回だけ

```shell
# リポジトリに移動して
$ cd your-repository
# aqua.yaml がなければ作る
# .pre-commit-config.yaml がなければ作る
# prek をテスト
$ prek run -a
# prek を hook として登録
$ prek install
```

あとは自動で処理されると思うので大丈夫
