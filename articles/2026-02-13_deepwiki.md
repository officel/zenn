# tl;dr

- [DeepWiki](https://deepwiki.com/) で自分の [dotfiles](https://deepwiki.com/officel/dotfiles) を見直してみた
- リポジトリを作ってる本人より詳しいドキュメントができててすごい

# DeepWiki

- [DeepWiki](https://deepwiki.com/) の解説はよそで
- リポジトリの内容からドキュメント化してくれるものって理解で充分
- OSS 等のリポジトリの理解を早めてくれる
- 無料で誰でも使えるしパブリックなリポジトリを追加することもできる
- プライベートリポジトリも無課金で追加できるらしいけど試していない（外に出したくないからプライベートリポジトリなわけで）

# はじまり

- [officel/github-web-cosmetic](https://github.com/officel/github-web-cosmetic) ってリポジトリを作っている
- Google Chrome の機能拡張で、DeepWiki へのリンクや画面の横幅をフルに使えるようにしてる
- ストアに出すのが面倒なので git clone してデベロッパーモードで自分で追加する必要がある
- で、先日これが壊れた（GitHub の WebUI の仕様変更っぽい）
- [Jules がしゅっと](https://github.com/officel/github-web-cosmetic/pull/37) 直してくれた
- 動作確認のために久しぶりに DeepWiki を見てて、追加していないリポジトリがあることに気がついた

# dotfiles

- [dotfiles リポジトリの式年遷宮](https://zenn.dev/raki/articles/2026-01-08_dotfiles) という記事を書いた
- 関連して [技術選定はツール比較ではなく破綻単位で考えるのがいいんじゃないかなって](https://zenn.dev/raki/articles/2026-02-08_select_tools) って記事も書いた
- そう [officel/dotfiles: My dotfiles.](https://github.com/officel/dotfiles) の DeepWiki 設定をしていなかった（のでやった）
- それがこれ [officel/dotfiles | DeepWiki](https://deepwiki.com/officel/dotfiles)
- リポジトリにあるファイルの関連や説明を図で表現してくれてめっちゃわかりやすい
- とくに bash まわりの XDG_BASE_DIRECTORY の説明は自分で説明するよりわかりやすくなってた
- フローチャートもわかりやすいけど、シーケンス図がとくにいい
- 感動して思わず記事にしたってワケ
