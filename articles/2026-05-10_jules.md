# tl;dr

- GW の宿題に、無課金AIでどの程度までもっていけるか試してみた
- ずぼらな性格と集中力のなさと完遂への決意が不足して中途半端になった
- 継続して育てていく予定だけど、とりあえず現状報告

# はじめに

- 個人的に、課金AIが強いのは当たり前だと思っていて、おもしろいとはあんまり思っていない
- 無課金が偉いとは思わないが、無課金でもここまでできる、は表現できる
- 課金の有無にかかわらず、AI に対する指示（プロンプト、コンテキスト、設計等のドキュメント）の出し方は勉強しておいたほうが後で楽なはず
- 日課でプレイしている Number Sums というスマホ向けパズルゲームがある
- 広告が多すぎて（プレイ前中後に入りまくる。必ずしも悪だとは言わないが）時間がもったいないって
- 昨今の AI であれこれ作りましたって多いし、業務では Terraform がメインなので、他の言語でのAI活用やってみるしか
- そんな感じで GW の宿題にパズルゲームを無課金AIで作ってみようと思ったってワケ

# というわけで

- 最初にみんな大好き [ChatGPT](https://chatgpt.com/) と壁打ちをして大枠を決めた

# 学び的なこと

- 作ってみた系でよくある MVP に集中、みたいなのが自分には向いてなかったｗ だってあれこれ積みたいし納得して次に行きたいから。。。
- 普段いじらない言語の勉強に AI はすごく便利
- 妥当性や取捨選択は [Google Gemini](https://gemini.google.com/app?hl=ja) のディープリサーチが役に立った
- Issue での指示だし（jules は Issue に assign-to-jules というタグを付けると勝手に読みに来てくれる）は、すごく便利
- ただし、指示内容が思ったとおりに伝わらないケースも数多くあって、人とIssueとドキュメントとコードで表現と統一する等の工夫は必要だと思った
- 設計の時点は考慮しなかったこと、細部や MVP 以降としたものなどについてをどう考えるかは人次第？つい拘ってしまって本筋ではないところに注力してしまったのは満足するゴールに届かなかった原因だと思う
- ぶっちゃけ昔書いた記事のとおり、問題を作成する基礎はあって広告を出さないプレイアブルなアプリができればよくて雑に始めてしまったせいで迷走したとは思う
- 設計とゴールにまい進する意志力みたいなの大事だなって
- 全体を常に把握している状態で作業して欲しいんだけど、指示に加えておかないとやらないことが多いので、そこらへんの調整が必要なうちは無理に AI がんばらなくてもいいかなって思ってる
- たとえば design.md に書いてないことの追加とか、変更が入った時に design.md を更新してって書いておくとやってくれるけど、書いてない時はやってくれないとか
- たとえばファイルAを修正したいって書いてしまうと、ファイルAだけでなんとかしようとして、共通系のファイルにある分を無視するとかその逆とか
- しょっちゅう pre-commit にひっかけるので、pre-commit を常に実行して欲しいんだけど指示に加えるのとか面倒だよね。
- リポジトリを読んで設定をひととおり読み込んだら、人の場合は対応しようとするじゃん。
- pnpm-lock.yaml があるなら pnpm 使うとか
- package.json の scripts に test が書いてあって、jest が devDependencies に書いてあれば npm run test くらいするとか
- コード系生成AIならエンジニアが普段当たり前に処理するようなところを当たり前に対応できるようになってて欲しいなって

# まとめ

- AIへの課金はお金持ってる人とか、そこに賭けたい人にがんばってもろて
- 大多数の普通のエンジニアは無課金でもできることがんばってもろて
- 課金の有無にかかわらず AI に任せるところ、自分でがんばるところの区別をつけられる程度には勉強しといてもろて
- とりあえずAI無課金でもパズルゲームアプリ作るくらいのことはできるんだと知ってもらえればいいかなって
- 個人的には夏休みくらいまでにもっとちゃんと詰めて、自分が満足できる形になったらストアにあげる、くらいのゆる目標でいいかなと
